## Prerequisites

Before you begin, ensure you have:

- **Python 3.11+** for agent development
- **An LLM provider** (Anthropic, OpenAI, Google Gemini, or local models via Ollama)
- **Git** for cloning the repository

<Note>
  **Windows Users**: Use WSL (Windows Subsystem for Linux) or Git Bash. Some core automation scripts may not execute correctly in standard Command Prompt or PowerShell.
</Note>

## Installation

Hive uses a `uv` workspace layout and is **not** installed with `pip install`. Running `pip install -e .` from the repository root will create a placeholder package and Hive will not function correctly.

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/aden-hive/hive.git
    cd hive
    ```
  </Step>

  <Step title="Run the quickstart script">
    ```bash
    ./quickstart.sh
    ```
    
    This automated script sets up:
    - **Framework** - Core agent runtime and graph executor (in `core/.venv`)
    - **Aden Tools** - 100+ MCP tools for agent capabilities (in `tools/.venv`)
    - **Credential store** - Encrypted API key storage (`~/.hive/credentials`)
    - **LLM provider** - Interactive default model configuration
    - All required Python dependencies with `uv`
  </Step>

  <Step title="Configure your LLM provider">
    The quickstart script will prompt you to configure your LLM provider. Enter your API key when prompted:

    ```
    ? Select your LLM provider:
      > Anthropic (Claude)
        OpenAI (GPT-4)
        Google (Gemini)
        Local (Ollama)
    ```

    Your credentials are stored securely in `~/.hive/credentials` with encryption.
  </Step>

  <Step title="Launch the dashboard">
    The script will automatically open the Hive dashboard in your browser at `http://localhost:3000`.

    <img src="https://github.com/user-attachments/assets/134d897f-5e75-4874-b00b-e0505f6b45c4" alt="Hive Dashboard" />
  </Step>
</Steps>

## Build your first agent

There are three ways to create agents in Hive:

<Tabs>
  <Tab title="Browser Dashboard">
    ### Using the Web Interface

    1. **Describe your agent**: Type what you want your agent to do in the input box on the home screen.
    
       <img src="https://github.com/user-attachments/assets/1ce19141-a78b-46f5-8d64-dbf987e048f4" alt="Agent Creation" />

    2. **The Queen Bee coding agent** will generate the agent graph, nodes, and connection code automatically.

    3. **Review and approve** the generated agent structure.

    4. **Run your agent** by clicking the Run button or asking the Queen Bee to execute it.

       <img src="https://github.com/user-attachments/assets/71c38206-2ad5-49aa-bde8-6698d0bc55f5" alt="Running Agent" />

    <Tip>
      Click "Try a sample agent" to explore pre-built templates. You can run them directly or customize them for your needs.
    </Tip>
  </Tab>

  <Tab title="Command Line (TUI)">
    ### Using the Terminal UI

    ```bash
    ./hive tui
    ```

    The TUI provides:
    - **Agent browser** - View all your agents
    - **Real-time execution** - Watch agents run with live updates
    - **Session management** - Resume paused sessions
    - **Log viewer** - Debug with full execution logs

    Navigate with arrow keys, press `Enter` to select, and `q` to quit.
  </Tab>

  <Tab title="Code-First">
    ### Programmatic Agent Creation

    Create an agent by defining its graph structure in Python:

    ```python
    from framework import Agent, Goal, SuccessCriterion
    from framework.graph import Node, Edge

    # Define your goal
    goal = Goal(
        id="research-report",
        name="Research Report Generator",
        success_criteria=[
            SuccessCriterion(
                id="comprehensive",
                description="Report covers all major aspects",
                metric="llm_judge",
                weight=0.5
            )
        ]
    )

    # Create nodes
    research_node = Node(
        id="research",
        type="event_loop",
        system_prompt="Research the topic thoroughly",
        tools=["web_search", "web_scrape"],
        outputs=["research_data"]
    )

    write_node = Node(
        id="write",
        type="event_loop",
        system_prompt="Write a comprehensive report",
        inputs=["research_data"],
        outputs=["report"]
    )

    # Define the agent
    agent = Agent(
        goal=goal,
        nodes=[research_node, write_node],
        edges=[
            Edge(source="research", target="write", condition="ON_SUCCESS")
        ]
    )

    # Run the agent
    result = agent.run({"topic": "AI agent frameworks"})
    print(result["report"])
    ```

    Save this as `my_agent.py` and run:
    ```bash
    ./hive run my_agent.py --input '{"topic": "AI agent frameworks"}'
    ```
  </Tab>
</Tabs>

## Run agents via CLI

Once an agent is created, you can run it from the command line:

```bash
# Run an agent
./hive run <agent-name> --input '{"key": "value"}'

# Run in interactive shell mode
./hive shell <agent-name>

# Show agent information
./hive info <agent-name>

# List all agents
./hive list
```

## Configuration

Your Hive installation creates several configuration files:

<Accordion title="~/.hive/config.yaml - Global settings">
  ```yaml
  llm:
    provider: anthropic
    model: claude-3-5-sonnet-20241022
    
  credentials:
    storage: encrypted_file
    
  logging:
    level: info
    runtime_logs: true
  ```
</Accordion>

<Accordion title="~/.hive/credentials - Encrypted credentials">
  ```
  # Auto-managed by the credential store
  # Use: ./hive credentials add <name>
  # View: ./hive credentials list
  ```
</Accordion>

<Accordion title="core/config.yaml - Framework configuration">
  ```yaml
  runtime:
    max_iterations: 10
    checkpoint_interval: 5
    
  monitoring:
    enable_health_check: true
    stream_events: true
    
  cost_tracking:
    enable: true
    budget_per_session: 5.0  # USD
  ```
</Accordion>

## Troubleshooting

<AccordionGroup>
  <Accordion title="'uv' command not found">
    The quickstart script installs `uv` automatically. If you see this error:

    ```bash
    # Install uv manually
    curl -LsSf https://astral.sh/uv/install.sh | sh
    
    # Add to PATH
    export PATH="$HOME/.local/bin:$PATH"
    
    # Verify installation
    uv --version
    ```
  </Accordion>

  <Accordion title="Port 3000 already in use">
    If another service is using port 3000:

    ```bash
    # Find the process
    lsof -i :3000
    
    # Kill it or change Hive's port
    PORT=8080 ./hive tui
    ```
  </Accordion>

  <Accordion title="LLM API key errors">
    Verify your credentials:

    ```bash
    # List stored credentials
    ./hive credentials list
    
    # Test a credential
    ./hive credentials test anthropic_api_key
    
    # Add or update
    ./hive credentials add anthropic_api_key
    ```
  </Accordion>

  <Accordion title="Python version issues">
    Hive requires Python 3.11+:

    ```bash
    # Check your version
    python --version
    
    # If using pyenv
    pyenv install 3.11
    pyenv local 3.11
    
    # Re-run quickstart
    ./quickstart.sh
    ```
  </Accordion>
</AccordionGroup>
